# Actividad 7 - Gestión de Proyectos con Git y GitHub

## Descripción del Proyecto
Este proyecto es una simulación de un flujo de trabajo colaborativo profesional. El objetivo principal es gestionar el ciclo de vida de un desarrollo web básico utilizando Git para el control de versiones local y GitHub como repositorio remoto. [cite_start]Se integran prácticas de creación de ramas, resolución de conflictos y uso de Pull Requests[cite: 7, 8].

## Funcionalidades Implementadas
* [cite_start]**Estructura Base:** Creación de archivos iniciales `index.html` y `README.md`[cite: 15, 109].
* [cite_start]**Barra de Navegación:** Implementación de un encabezado funcional en la rama `feature/navbar`[cite: 22, 282].
* [cite_start]**Pie de Página:** Agregado de un footer mediante flujo de trabajo de Pull Request en la rama `feature/footer`[cite: 32, 393].
* [cite_start]**Sincronización:** Vinculación exitosa entre repositorio local y remoto con resolución manual de conflictos[cite: 147, 358].

## Ramas Utilizadas
* [cite_start]**main / master:** Rama principal donde se integra el código estable[cite: 28, 147].
* [cite_start]**feature/navbar:** Utilizada para el desarrollo de la barra de navegación[cite: 22, 281].
* [cite_start]**feature/footer:** Utilizada para simular la colaboración mediante Pull Request en GitHub[cite: 32, 393].
